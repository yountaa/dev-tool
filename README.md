# dev-tool

Модульный инструмент для эксплуатации. Модуль **silences** ставит алерты на
silence в Alertmanager — разово по датам и по расписанию. Бэкенд (FastAPI) и
фронт (Vue 3 + Vite) — отдельные сервисы, разворачиваются независимо.

## Переменные — бэкенд


| Переменная | По умолчанию | Описание                                                                                                                                                                  |
| -------------------- | ----------------------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `alert_<имя>`     | —                      | **обязательно** (хотя бы одна). URL Alertmanager, каждая = вкладка окружения. Напр. `alert_prod=http://alertmanager.prod.svc:9093` |
| `PORT`               | `8000`                  | порт API                                                                                                                                                                      |
| `GIT_REPO_URL`       | —                      | HTTPS-адрес репы-хранилища конфигов                                                                                                                     |
| `GIT_USER`           | —                      | логин для git                                                                                                                                                             |
| `GIT_TOKEN`          | —                      | deploy-токен                                                                                                                                                                 |
| `GIT_BRANCH`         | `main`                  | ветка                                                                                                                                                                        |
| `GIT_LOCAL_DIR`      | `./.hub-repo`           | папка локального клона репы                                                                                                                               |
| `SILENCE_CRON`       | `*/5 * * * *`           | как часто шедулер проверяет расписания (cron)                                                                                                   |
| `SILENCE_TZ`         | `Europe/Moscow`         | таймзона окон расписания                                                                                                                                    |

`GIT_*` нужны для режима «по расписанию» и истории. Без них шедулер выключен,
а разовый silence в AM поставить всё равно можно.

## Переменные — фронтенд


| Переменная | По умолчанию | Описание                                                                                                                    |
| -------------------- | ----------------------- | ----------------------------------------------------------------------------------------------------------------------------------- |
| `BACKEND_URL`        | `http://backend:8000`   | **адрес бэкенда** — куда внутренний nginx проксирует `/silences` и `/health`                  |
| `FRONTEND_PORT`      | `80`                    | порт, который слушает nginx внутри контейнера                                                     |
| `VITE_API_BASE`      | пусто              | build-arg (не runtime). Только если фронт должен ходить в API напрямую, мимо прокси |

К какому бэкенду подключаться, задаёт `BACKEND_URL` (в проде). В dev-режиме
(`npm run dev`) адрес бэка берётся из `server.proxy` в `frontend/vite.config.js`.
Фронт всегда зовёт относительные пути `/silences/...`, адрес бэка в коде не зашит.

## Порты


| Сервис | По умолчанию | Переопределить |
| ------------ | ----------------------- | ---------------------------- |
| backend      | `8000`                  | `PORT`                       |
| frontend     | `80`                    | `FRONTEND_PORT`              |

## Запуск

Локальный стенд для проверки (фронт + бэк + тестовый Alertmanager) — `docker-compose.yml`
в корне, переменные в `.env`. Прод собирается и деплоится сервисами по отдельности
из `backend/` и `frontend/`.
